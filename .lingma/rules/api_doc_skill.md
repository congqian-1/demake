**输出要求**：
严格按照以下结构和顺序输出，不得更改顺序或跳过任何章节：
1. 接口描述
2. 前提条件
3. 使用限制
4. 实现方式简介
5. 流程图
6. 接口说明
7. 请求
8. 响应

**限制**：
1. Markdown格式
2. 禁止插入任何空行，所有内容必须连续排列，段落、列表、标题之间均不得换行留空
3. 列表和表格的表头以及单元格大小按照内容适配

- 不得跳过任何部分

1. **URL 路径**：只写相对路径，不包含域名（如：`/marketas-api/partner-task/saveTaskOption`）

3. **子对象说明**：复杂对象类型需单独列表说明字段结构
4. **请求示例**：为每个重要业务场景提供独立示例，并注明必传参数
5. **响应结构**：如有分页，先说明分页字段，再说明数据字段
6. **错误码**：至少包含常见的 4xx、5xx 错误
7. **代码块语言**：JSON 示例使用 `json` 标记，Shell 示例使用 `bash` 标记
